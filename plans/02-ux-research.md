# UX 리서치 - 유전체 데이터 웹 플랫폼 분석

## 1. 벤치마킹 대상

### 1.1 Phytozome (JGI)
- **URL**: https://phytozome-next.jgi.doe.gov/
- **특징**: 250+ 식물 유전체 호스팅
- **네비게이션**: 상단 메뉴 바 (Home, Search, BLAST, BioMart, Data Portal)
- **검색**: 키워드 검색 + BLAST/BLAT 시퀀스 유사성 검색
- **데이터 접근**: BioMart 데이터 웨어하우스로 테이블 형태 추출 (HTML, TSV, CSV, XLS)
- **시각화**: Genome Browser로 유전자 위치 시각화
- **v13 개선점**: 더 직관적인 인터페이스, 유지보수 용이한 인프라

**우리 프로젝트에 적용할 점**:
- 상단 메뉴를 3-5개로 제한 (심플하게)
- 데이터 다운로드 시 포맷 선택 옵션 제공

### 1.2 TAIR (The Arabidopsis Information Resource)
- **URL**: https://www.arabidopsis.org/
- **특징**: 애기장대 모델 식물 종합 DB
- **검색**: 심플 검색 + 고급 검색 (250개 이름 동시 검색), 벌크 다운로드
- **시각화**: SeqViewer (인터랙티브 게놈 뷰어), GBrowse
- **데이터 구조**: 유전자, 단백질, 변이, 표현형, 마커, 물리/유전 지도
- **큐레이션**: PhD 연구자가 직접 큐레이션

**우리 프로젝트에 적용할 점**:
- 심플 검색 + 고급 검색 분리
- SeqViewer 스타일의 인터랙티브 뷰어는 추후 Phase로 분리

### 1.3 Ensembl Plants
- **URL**: https://plants.ensembl.org/
- **특징**: 대규모 식물 유전체 통합 브라우저
- **네비게이션**: 종 선택 -> Location/Gene/Variation 탭 구조
- **BioMart**: 프로그래밍 없이 데이터 마이닝, 필터+속성 선택 -> 테이블 다운로드
- **시각화**: Karyotype 뷰, Location 탭에서 게놈 스크롤 브라우징
- **BLAST**: 개별 종 또는 최대 25종 동시 검색

**우리 프로젝트에 적용할 점**:
- 탭 기반 정보 구조 (품종 상세 페이지에서)
- 필터 + 속성 선택 방식의 데이터 내보내기

### 1.4 GBIF (Global Biodiversity Information Facility)
- **URL**: https://www.gbif.org/
- **특징**: 생물다양성 발생 데이터, 지도 기반 인터페이스
- **지도**: Leaflet + OpenStreetMap 기반 인터랙티브 맵
- **마커**: 클릭 시 메타데이터 팝업, 마커 클러스터링
- **Maps API**: 타일 기반 웹 맵 서비스

**우리 프로젝트에 적용할 점**:
- GBIF 스타일의 지도 인터페이스 참고
- 마커 클릭 -> 팝업 -> 상세 페이지 이동 패턴
- Leaflet + OpenStreetMap 조합 (무료, 경량)

## 2. UX 패턴 분석

### 2.1 네비게이션 구조
| 패턴 | 설명 | 적용 |
|------|------|------|
| 상단 고정 메뉴 | 3-5개 메인 항목 | O - 홈, 품종, 지도, 다운로드, About |
| 브레드크럼 | 깊은 계층 탐색 시 위치 표시 | O - 품종 상세 페이지 |
| 사이드바 필터 | 데이터 테이블 옆 필터 패널 | O - 품종 목록 페이지 |
| 탭 네비게이션 | 상세 정보 분류 | O - 품종 상세 (기본정보/유전체/파일) |

### 2.2 데이터 검색/필터
- **심플 검색**: 상단 검색바 (글로벌 검색)
- **고급 필터**: 사이드 패널에 종명, 수집지, 날짜 등 필터
- **자동완성**: 품종명/종명 입력 시 자동완성 제안
- **벌크 검색**: 여러 ID 동시 검색 (textarea 입력)

### 2.3 데이터 테이블
- **정렬**: 컬럼 헤더 클릭으로 정렬
- **페이지네이션**: 하단 페이지 네비게이션 (10/25/50/100 per page)
- **컬럼 토글**: 표시할 컬럼 선택 가능
- **행 클릭**: 클릭 시 상세 페이지 이동
- **내보내기**: CSV/TSV/Excel 다운로드 버튼

### 2.4 지도/위치 정보 표시
- **메인 지도**: Leaflet + OpenStreetMap 타일
- **마커 클러스터링**: 줌 레벨에 따라 마커 그룹화
- **팝업**: 마커 클릭 시 간략 정보 + 상세 링크
- **미니맵**: 상세 페이지에서 수집 위치 표시용 작은 지도
- **테이블 연동**: 테이블 행 hover 시 지도 마커 하이라이트

### 2.5 파일 다운로드
- **파일 목록**: 파일명, 크기, 형식, 업로드 날짜 표시
- **카테고리 필터**: 파일 형식별 (FASTA, VCF, GFF 등)
- **벌크 다운로드**: 체크박스 선택 후 일괄 다운로드
- **직접 링크**: 각 파일에 직접 다운로드 링크

## 3. 바이오테크 웹 디자인 Best Practice

### 3.1 일반 원칙
- **복잡한 과학 -> 접근 가능한 스토리텔링**: 비전문가도 이해할 수 있는 UI
- **메뉴 항목 3-5개 제한**: 사용자 과부하 방지
- **일관성**: 동일한 인터랙션 패턴 반복 사용
- **사용자 여정 매핑**: 연구 -> 평가 -> 다운로드 흐름 설계

### 3.2 식물 유전체 DB 특화
- **종/품종 중심 네비게이션**: 종 선택이 첫 진입점
- **데이터 형식 다양성 대응**: FASTA, VCF, GFF3 등 여러 포맷
- **대용량 데이터 처리**: 점진적 로딩, 페이지네이션 필수
- **시각화 우선**: 테이블만으로는 부족, 차트/지도 병행

## 4. 우리 프로젝트 적용 요약

### 핵심 UX 원칙
1. **심플한 네비게이션**: 5개 메뉴 (홈, 품종, 지도, 다운로드, About)
2. **지도-테이블 연동**: 품종 목록에서 테이블과 지도가 실시간 연동
3. **점진적 공개**: 목록 -> 상세 -> 파일 순으로 정보 깊이 증가
4. **검색 우선**: 상단 글로벌 검색바로 빠른 접근
5. **다운로드 편의**: 파일 형식 필터 + 벌크 다운로드
6. **반응형**: 모바일에서는 지도/테이블 세로 스택

### 차별화 포인트
- GBIF 스타일의 **지도 중심 탐색** (기존 유전체 DB는 텍스트 중심)
- **비로그인 접근** (대부분의 유전체 DB는 로그인 요구)
- **심플한 인터페이스** (Phytozome/Ensembl 대비 학습 곡선 최소화)
